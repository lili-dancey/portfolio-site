WiseWallet

WiseWallet was a conceptual project created as a solution for users who are looking to save money for a particular goal. The design process began by understanding the user, creating user flows and through user testing, the main features and screens of the app were created.

  • Project Type - Conceptual project during CareerFoundry course
  • Platform - Mobile App
  • My Role - UX, UI Designer
  • Timeline - 13 days, 14 screens
  • Tools - Figma, Procreate, Google Meet

My process

User flows

I chose to create a flow that had two sections to it. The first is the onboarding process. This included a sequence of screens that would follow when a user creates an account. The second is the set of screens with which a user would interact after creating an account or viewing their home, dashboard and spending insights.

Low-fidelity wireframes

Using the user flows, I created preliminary screens for both scenarios.

Mid-Fidelity Wireframes

The low-fidelity wireframes were adapted to represent a more accurate representation the information and layout without losing focus on colour and styling.

Scenario user testing

To understand user preferences, pain points, and suggestions for improvement. Users were presented with the mid-fidelity wireframes and asked to imagine themselves using the app in two different scenarios.

Scenario 1

You are a new user and you want to sign up and set up your account information.

Scenario 2

You have logged into your WiseWallet app and would like to view your dashboard and spending insights as well as add an income.

Insights
  1. Give users an insight on how to achieve their savings goal.
  2. Simple and clear layout helps make the user feel trustful of the website.
  3. The use of charts makes the information easier to understand.

Impact

Provided insight into how comfortable users feel to use the features required in the scenario as well as monitoring how instinctual they find the user journey mapping, layout and design.

Adaptations

The user testing provided useful insight into how the screens could be adapted in order to obtain optimal results.

Users requested more information about their savings goal, their progress and advice to help expedite the savings process.

Adding the option to review their monthly balance.

Main features

As part of the design brief, user stories were provided. These stories formed the basis of each screen’s content and layout. Using iterative design techniques, the app’s main feature screens were created.

Personalised spending insights

Based on a users daily spending and costs, a personalised list of methods with which a user can potentially cut costs will be presented.

Select goal

The onboarding process will require the user to enter necessary information about their savings goal, the purpose and provide information on their desired timeline and the amount they have already saved.

Monthly savings

Users will then be prompted to enter a monthly recurring transfer amount.

Dashboard view

A dashboard view will provide users with a clear visual about their financial information in a variety of ways, with the option of accessing more detailed information.

Desktop perspective

The desktop version of the dashboard holds more functions - an overview of the user’s progress towards their savings goal, a breakdown of their income sources, and more information about their profile and preferences.

Final screens

Conclusion

Designing with finances in mind requires a careful balance between detailed information and simple and concise language and layout. Therefore brand guidelines were imperative to follow. The ideas behind creating this app were carefully considered and as a designer, it is essential that what is portrayed mirrors the initial idea and concept.
